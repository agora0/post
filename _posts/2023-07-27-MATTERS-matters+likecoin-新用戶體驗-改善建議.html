第一次接觸 Matters 的新用戶也許會對如何搭配使用 Liker ID 操作感到相當陌生並產生各種問題，以下提出一些提升用戶體驗的介面細節修改建議，大部份只是改一改過時的字眼和連結的小修改，部份已直接發出 pull request 到 Matters 開源庫，有興趣的用戶歡迎參與跟進討論。

一）改善「設置 Liker ID」 彈出式視窗

新 Matters 用戶會在四個情境中，於「設置 Liker ID」彈出式視窗中第一次接觸有關 “Liker ID” 的資訊：

1. 點擊站上的 LikeCoin button 讚賞鍵

2. 進行新手任務

3. 在「設定」選擇「錢包設定」中的「設置 Liker ID」

4. 在文章點擊「支持作者」，選擇以 LikeCoin 支持，再被引導至「設定」頁

這是非常重要的使用者介面，幾乎是連接所有 Matters 新用戶到 Liker Land 的必經之路。然而現時有幾點不足之處：

1. 不能假設新用戶完全明白什麼是 LikeCoin、讚賞公民或 Liker ID，故此他們很可能會點擊「了解更多 help.like.co」試圖去理解更多。然而該鏈結是連接至英語的 LikeCoin 文檔首頁，文檔是個龐大的資料庫，把用戶丟到首頁會頓時令用戶被大量資訊淹沒而感到不知所措， 也令習慣閱讀中文的新用戶感卻步。

建議：鏈結至更為簡單易明的 LikeCoin 文檔頁面，例如 https://docs.like.co/v/zh/user-guide/creator/matters。

2. 用戶對「生成 Liker ID」和「綁定 Liker ID」的選項感到困惑，不明白兩者有何分別。

建議：「生成」和「綁定」是技術性描述，用戶難明所以；對新用戶而言，比較直接的用語建議為「創建新的 Liker ID」及「綁定現有 Liker ID」，讓用戶立即按自身現有的狀況判斷如何選擇。

3. 新用戶很可能剛在接觸此介面的幾分鐘之前才創建了「Matters ID」，於是容易跟現在的「Liker ID」搞混。

建議：加入類似「注意：Matters ID 跟 Liker ID 是不同、獨立的 ID」的註解。

二）在 Matters 查看 Liker ID 或 LikeCoin 錢包詳情

目前於 Matters 介面無法查詢到有關 Liker ID 的詳細資訊，例如 LikeCoin 錢包地址。新用戶因此必須離開系統並自行於其他渠道搜尋，例如 Google 「LikeCoin」、「Liker ID」、「讚賞公民」等關鍵字，然後卻很可能讀了跟流程不連貫的資訊而感到迷惘。

另一方面，就算用戶能夠到達正確的設定頁面，亦即是 https://like.co/in 。由於頁面已把經 Authcore 登入的舊介面藏到較隱蔽的位置，用戶或會找不到「以 Matters 登入」的社交帳號登入選項。

建議：

1. 把「錢包設定」下的 Liker ID 欄位改成可供點擊，並鏈結至 https://like.co/in。

2. 經 Matters 設定頁連結至 https://like.co/in 的網址中加入「查詢字串」如 “?from=matters"，令 like.co 能夠辨識這個特別情境，並顯示 Authcore 登入介面（Matters 為其中一個登入選項）。

3. 將 Matters 登入選項優先排序於 Twitter、Facebook 等社交帳號登入之先。

三）更新有關 ISCN 的描述

「註冊 ISCN」的描述鏈結現時是指向 ISCN 的技術文檔 https://iscn.io，對一般非技術背景的用戶而言並不相關。另，用戶註冊 ISCN 的最終用途是發行 Writing NFT，所以不妨加入一句提示，讓用戶了解「為什麼」。

建議：

1. 把鏈結改為指向創作者關心的內容： https://docs.like.co/v/zh/general-guides/writing-nft/nft-portal#publish-writing-nft-with-iscn-id

2. 把描述改成 「⋯⋯用以出版 Writing NFT。費用約 1 LIKE，推廣期由平台補助。」（並刪除「發布時註冊 ISCN」，因為「註冊 ISCN」的標題本身經已包含了相同意思，可保持文案簡潔。）

四）改善對接 https://app.like.co 的體驗

當創作者完成註冊 ISCN 後，若點擊 Matters 文章中「分佈式入口」中的 ISCN 鏈結並被引導至 app.like.co，會立即感到迷失，因為在 app.like.co 是沒有辦法使用 Matters 登入的 - 系統沒有提供 authcore 登入介面。

更為尷尬的是，假若用戶使用 Matters 生成 Liker ID，他們必須要匯出助記詞並導入 Keplr 以便使用相同的錢包去發行 Writing NFT - 這䌓複的步驟會令用戶卻步。

建議：沒有速成解決方案。Liker Land 或可考慮在 app.like.co 繼續提供如 https://like.co/in 般的 authcore 登入介面，又或者起碼提供超鏈結以指引用戶匯出及導入助記詞。

然而，終極解決方案或許是使用 Liker Land app 透過 Wallet Connect 登入 Matters。

未來展望：以錢包和 Wallet Connect 為主的去中心化登入

Matters 和 Liker ID 的功能整合已有 4 年歷史，這段時間中 Web3 科技和社群經歷了大起大落的發展和變化。LikeCoin 已由當初 ERC20 Token 轉變成為了 cosmos-sdk 生態的 IBC Token，Matters 開始支援以 MetaMask 錢包登入帳號，並舉行了很多以 EVM 技術為基礎的試驗性活動。但目前 Matters 和 LikeCoin 的鏈接，還是沿用當初 Web2.0 的 oAuth 技術，以服務器互相認證為主，留下了複雜的技術債務和 UX 問題。 隨着 Web3 技術發展 Wallet Connect 技術開始成熟，而 MetaMask 也即將以 snap 支援 Cosmos 生態的錢包操作，在不遠的將來可預期 Matters 和 LikeCoin 的整合轉變成以錢包為基礎。複雜的跳轉認證流程將可以化為簡單一個簽名，讓用户以友善簡化流程更輕鬆地進入寫作的 Web3 宇宙。
